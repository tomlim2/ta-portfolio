동기

PMX→VRM 변환 파이프라인을 만들면서, 변환 전에 원본 모델과 모션이 제대로 동작하는지 확인할 도구가 필요했다. 임포트가 안 될 때 PMX 원본 탓인지 VRM 컨버터 탓인지 가려내는 게 핵심 — PMX 자체에 텍스처 누락이나 본 결여가 있다면 컨버터 문제가 아니다.

사용자: 기획자, QA — TA뿐 아니라 비개발 직군이 직접 열어서 확인.

해결

모션 호환성 자동 분석

VMD 모션 파일을 분석하여 PMX 모델과 본 매칭률을 계산하고, IK 충돌과 소스 모델 계열을 자동 감지. 게임사 PMX도 이 계열을 따르는데 — 예: 스타레일은 DIVA 계열, 원신은 ミリシタ 계열.

체형 자동 보정 (IK 사이징)

모델 간 힙 높이 비율로 IK 포지션을 자동 스케일링. 소스 모델이 없어도 VMD 애니메이션에서 바닥 접촉 프레임의 중앙값으로 원본 체형을 추정한다.

체형 보정 Before/After — IK 사이징 적용 전후 비교 (발 부유 → 바닥 접촉)

모션 연동 이펙트 — Precompute 실험

VMD 모션 데이터에서 벡터 필드 등 CPU 기반 효과를 사전 계산하면 어떨지 실험. 결과가 매우 좋았다 — 가볍고, 모션 트랙만 있으면 이펙트 타이밍이 자동으로 잡힌다. 기획팀과 클라이언트팀에게 "잘 움직인다"는 것을 보여주기 위한 POC.

6종: Foot Ripple, Spark Burst, Ground Reflect, Falling/Rising Light, Velocity Effect

기술 상세

VMD Precompute — 이펙트 사전 계산
모션 데이터에서 블룸 이벤트, 발 착지 타이밍, 각속도 피크를 사전 계산. 실시간 분석 대신 precompute 방식으로 프레임 드롭 없이 이펙트를 동기화.
IK 사이징 — 소스 모델 없이 체형 추정
타겟 모델의 rest pose에서 IK Y 위치를 기준으로 ground contact 프레임을 감지. 해당 프레임의 센터 Y 중앙값으로 소스 모델의 힙 높이를 추정하고, targetHipY / sourceHipY 비율로 IK/센터/グルーブ의 Y 포지션을 스케일링.
TSL 셰이더 — WebGPU 커스텀 머티리얼
Three.js TSL(Three Shading Language) 기반 커스텀 머티리얼. Ground Mirror Reflection, 이펙트 파티클 등을 TSL 노드로 구현. WebGPU의 차세대 렌더링 파이프라인을 직접 검증.
VMD 바이너리 파서 + 호환성 분석
VMD 파일에서 본 키프레임, 모프 키프레임, IK 키프레임을 파싱. PMX 모델의 본 트리와 대조하여 매칭률, 누락 본, IK 충돌을 자동 리포트.
