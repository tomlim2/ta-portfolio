PMX to VRM Pipeline

커뮤니티 PMX 캐릭터를 UE에서 바로 쓸 수 있도록 — 물리, 본, 머티리얼을 보존하는 자동 변환 파이프라인
Pipeline Architecture, Tool Development 2026년 2월~ (35 커밋) 회사 프로젝트
Python TypeScript glTF 2.0 VRM 0.x Next.js Web Worker

문제

Booth/커뮤니티에서 배포되는 PMX 캐릭터를 프로젝트에 투입해야 했지만, PMX를 UE에서 쓸 수 있는 VRM으로 변환할 방법이 없었다. 수동으로 옮기면 본 매핑이 깨지고, 머리카락/스커트 등 물리가 날아가며, 텍스처가 깨져서 룩이 완전히 달라졌다.

해결

PMX → glTF → VRM 자동 변환 파이프라인. ZIP을 넣으면 UE에서 바로 쓸 수 있는 VRM이 나온다. 물리(머리카락, 스커트, 리본)와 머티리얼 룩이 깨지지 않는 게 핵심이었다.

파이프라인 흐름

ZIP → scan .pmx → humanoid check (16 bones) → readPmx() → buildGltf() → buildVrm() → writeGlb() → renameVrm() → validate()
도구 사용자 방식
Python CLITA / 배치 작업여러 모델을 한 번에 변환
Next.js 웹앱팀 내부 누구나브라우저에서 ZIP 업로드 → VRM 다운로드
Browser SPA외부 / 서버 없이클라이언트 단독 변환

결과

항목 결과
변환율주요 게임사 PMX 90% 성공 (호요버스, 이터널리턴, 사이퍼즈 등)
모델 지원일본 + 중국 양쪽 PMX 커버
변환 속도393ms (Saber.pmx, 28K vertices)

왼쪽: 원본 게임 스크린샷 — 오른쪽: PMX → VRM 변환 후 뷰어 렌더링

기술 상세

물리 변환

스프링본 변환 (리지드바디+조인트 → VRM SpringBone)
PMX의 Bullet Physics 기반 리지드바디+조인트를 VRM의 스프링본으로 변환. 짧은 체인(리본, 액세서리)은 거의 고정, 회전 제한이 완전 잠긴 체인은 스킵, 중국 PMX의 과도한 dragForce 캡핑 등 모델별 예외 처리.

검증

6-Layer VRM 밸리데이터
GLB 컨테이너 → glTF 스펙 준수 → VRM 확장 구조 → 휴머노이드 본 완성도 → 스프링본 애니메이션 → 머티리얼 호환성. 각 레이어를 독립 검증해서 변환 안 되는 모델은 어디서 막혔는지 바로 알려준다.
CJK 인코딩 라운드트립
PMX 파일의 텍스처 경로, 본 이름 등이 Shift-JIS, GBK, EUC-KR, Big5 중 어떤 인코딩인지 자동 판별. Fallback 체인으로 모지바케 없이 디코딩해서 텍스처가 빠지는 걸 막는다.
