Character System

300개+ 캐릭터를 툴 로딩 6초로 — 파츠 조합부터 프리뷰, 외부 VRM까지
Tool Design, C++ Plugin, UE Integration 2024~2026 회사 프로젝트
Unreal Engine 5 C++ Blueprint Data Asset VRM4U Unreal Insights

문제

캐릭터가 300개 가까이 늘면서 파츠 조합부터 등록, 프리뷰, 로딩까지 체계적으로 관리할 파이프라인이 필요했다. 에디터를 열면 전체 에셋이 하드 레퍼런스로 올라와 30초를 기다려야 했고, 색감이나 애니메이션을 확인하려면 매번 게임을 실행해야 했다. 외부 VRM까지 지원하게 되면서 모델마다 카메라와 썸네일을 따로 잡아주는 수작업도 생겼다.

해결

에디터(EnvCheck), 런타임(Basemodel_TF), VRM Reference Points — 세 독립 시스템이 툴의 데이터 에셋 및 외부 모델 로드를 자사 셰이더 시스템과 애니메이션을 함께 확인할 수 있게 설계했다. 아티스트가 에디터에서 파츠를 조합하고 프리셋을 등록하면 Data Asset이 생산되고, 런타임이 그걸 비동기로 로드하는 흐름이다.

BP_Basemodel_TF 하나에 세이브와 로드가 같이 들어 있었는데, 세이브 쪽이 물고 있는 레퍼런스 때문에 에디터를 열 때마다 30초씩 걸렸다. 세이브를 BP_Character_EnvCheck로 빼고 Basemodel_TF는 로드만 남겨서 30초→6초로 줄였다. EnvCheck는 에디터 Tick Interface도 붙여서 게임 실행 없이 배경 위에서 바로 프리뷰할 수 있다. 외부 VRM은 Reference Points가 4단 폴백으로 얼굴을 찾아서 카메라 타겟과 썸네일까지 자동 처리.

시스템 구조

세 독립 코드베이스가 Data Asset과 VRM 정보 노드로 연결되는 구조. 에디터(EnvCheck)가 Data Asset을 생산하면 런타임(Basemodel_TF)이 비동기로 로드하고, VRM Reference Points가 외부 모델의 정보를 런타임에 업데이트한다.

에디터 — BP_Character_EnvCheck

아티스트가 에디터에서 머리·헤어·바디 파츠를 조합하고 프리셋으로 저장. 에디터 Tick Interface로 게임을 실행하지 않아도 배경 위에서 캐릭터와 애니메이션을 바로 확인할 수 있다. 이 과정에서 생산된 Data Asset이 런타임의 입력이 된다.

런타임 — BP_Basemodel_TF

EnvCheck와는 별도 코드베이스로, CinevCharacter를 상속받는 런타임 캐릭터 BP. 자체 캐릭터는 EnvCheck가 생산한 Data Asset에서 파츠별로 비동기 로드(Head/Hair/Body 분리), 외부 VRM은 로더를 통해 통째로 로드. 같은 BP에서 처리하되 로드 경로만 분기하여 300개+ 자체 프리셋과 커뮤니티 캐릭터를 하나의 시스템으로 관리한다.

외부 모델 지원 — VRM Reference Points

VRM4U 플러그인에 작성한 C++ 모듈로, BP_Basemodel_TF의 VRM 정보 노드를 업데이트하는 역할. 외부 유저가 가져온 어떤 VRM/PMX든 4단 폴백으로 얼굴 위치를 자동 탐지하고, 그 결과를 카메라 타겟으로 사용하여 얼굴 클로즈업·상반신·전신 3앵글 썸네일을 자동 촬영한다. 커맨드렛 환경을 셋팅하여 에디터 없이 배치로 전체 캐릭터 썸네일을 일괄 생성할 수 있다. A포즈/T포즈도 자동 판별.

기술 상세

로딩 30초→6초

Soft Object Reference + Async Loading
30초의 원인은 두 가지. 세이브용 파츠별 Hard Reference가 에디터 로드 시 전부 올라온 것, 그리고 80개 넘는 미사용 블렌드쉐이프(얼굴 형태)도 같이 딸려온 것. 세이브를 EnvCheck로 분리해서 Reference들을 전부 덜어냈고, 블렌드쉐이프도 미리 안 올리게 바꿨다. 로드할 때만 필요한 Data Asset을 StreamableManager.RequestAsyncLoad로 비동기 처리.
Data Asset Only 전환
메타휴먼 프리셋 구조에서 힌트를 얻었다. DataTable은 소프트 레퍼런스를 써도 첫 로드 시 전체 행을 다 읽어야 해서 근본적으로 느리다. Data Asset은 필요한 것만 소프트 레퍼런스로 들고 있으면 되니까 구조 자체가 가볍다.
Unreal Insights 프로파일링
CPU Timing 탭과 Loading 탭에서 LoadObject 콜스택을 추적해서 어떤 에셋이 시간을 먹고 있는지 찍어봤다. 이 데이터가 Hard Reference를 걷어내는 판단 근거.

캐릭터 액터 BP 구조

일체형 / 분리형 스켈레탈 메시 로드 구조
분리형(자체 캐릭터)은 Data Asset에서 Head/Hair/Body 파츠를 따로따로 로드하고, 일체형(VRM/PMX)은 VRM4U 로더로 모델 통째로 올린다. 같은 BP에서 로드 경로만 갈라지고, 로드되면 스켈레탈 메시 컴포넌트를 전부 리셋해서 다시 붙인다.
에디터 Tick Interface — PIE 없이 에디터 프리뷰 (BP_Character_EnvCheck)
BP_Character_EnvCheck에 에디터 Tick Blueprint Interface를 구현하여 에디터 환경에서도 애니메이션 재생 가능. 아트 디렉터가 런타임 실행 없이 색감, 크기감, 애니메이션을 바로 확인 가능하여 피드백 루프 단축.

얼굴 자동 탐지

VRM Reference Points 4단계 폴백
외부 VRM/PMX 캐릭터의 얼굴 랜드마크를 자동 탐지. 1차: Bone lookup (humanoidBoneTable). 2차: Material vertices + raycast (Moller-Trumbore). 3차: Position interpolation. 4차: Fallback estimation. 탐지 결과를 카메라 타겟과 썸네일 자동 촬영에 활용.
